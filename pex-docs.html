PEX Plugin Documentation

About the PEX Plugin

The PEX plugin is used to replace the default SIP shift register.

The plugin includes the folowing files:

  • pex.py
    The python program controls the ports. Location: SIP/plugins

  • pex.html
    A web.py template that provides a user interface for the plugin. Two dictionaries are used to pass and syncronize data between the UI and the PEX runtime environment: pex.pex_conf, gv. Location: SIP/templates

  • pex-docs.html
    This file. Provides documentation for the pex plugin including any special hardware needed and set up instructions. Location: SIP/static/docs/plugins

  • pex.manifest
    A text file containing a list of each of the plugin’s files with the path to it’s location. The manifest is used by the plugin manager to install and uninstall the plugin. Location: SIP/plugins/manifest

  • pex.json
    Configuration file containing persistant storage of port extender settings. It is created by the plugin the first time it is run and is not installed with the other files. Location: SIP/data

Using the pex plugin

This plugin is disabled by default. If you are reading this, the pex plugin is probably enabled on your system. Otherwise you can enable it using the plugin manager or by setting the group permissions of the pex.py file to executable. Once it is enabled, and the SIP program has been restarted, You can verify it is working by selecting it from the PLUGINS drop down menu of the main program’s web interface. You should be taken to the plugin’s Settings page.
